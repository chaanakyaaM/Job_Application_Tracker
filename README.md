# 📧 Job Application Tracker

A self-hosted app that automatically scans your Gmail and tracks all your job applications in one place.

---

## What it does

- Scans your Gmail for job-related emails using a smart keyword filter
- Automatically categorizes each email as Applied, Assessment, Interview, Offer, or Rejected
- Shows a dashboard with metrics, filters, search, and a pie chart breakdown
- Sends a one-time password (OTP) to your Gmail before showing the dashboard (is definitly a overkill for self-hosted application but it is fun)so that only you can access your job application mails
- Lets you download your applications as a CSV file

## How the email filter works

Every email goes through a layered filter before appearing in your dashboard:

1. **Sender denylist** — rejects Reddit, social media, etc. immediately
2. **Exclude phrases** — rejects marketing, newsletters, job alert digests
3. **Strong signal phrases** — trusted unconditionally (e.g. "thank you for applying")
4. **Weak signal phrases** — trusted only when paired with a job context keyword
5. **ATS domain fallback** — emails from known hiring platforms get extra trust

Status is detected in priority order: Rejected → Offer → Interview → Assessment → Applied.

## Learning

- **Google OAuth2 flow** — how `credentials.json` and `token.json` work, how refresh tokens silently renew access without re-authenticating
- **Gmail API** — searching emails with advanced queries, pagination, parsing MIME payloads to extract plain text and HTML bodies
- **OTP system from scratch** — generating cryptographically secure random codes with the `secrets` module (safer than `random`), expiry with `datetime + timedelta`, single-use clearing after verification
- **Timing-safe comparison** — why `hmac.compare_digest` is used instead of `==` to prevent timing attacks
- **smtplib + Gmail App Password** — sending email programmatically without exposing your real Gmail password
- **FastAPI** — building REST endpoints, query params, request body parsing with `Body(...)`, dependency injection with `Depends`
- **Rate limiting with slowapi** — protecting endpoints from abuse, why `request: Request` is required in the function signature
- **Negation-aware phrase matching** — detecting "not moving forward" correctly without false-positiving on "will not be moving forward" appearing in a negated sentence

## Notes

- `token.json` and `credentials.json` should never be committed to a public repo — add them to `.gitignore`
- The OTP expires after 10 minutes and is single-use
- Gmail is scanned up to 180 days back by default
- On Render, the filesystem is ephemeral — always pass `token.json` as an environment variable
